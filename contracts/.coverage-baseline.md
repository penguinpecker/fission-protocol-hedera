## Production targets (gating mainnet)

- ≥ 90 % branch on `core/` + `libraries/`
- ≥ 85 % branch on `sy/` + `periphery/`
- ≥ 85 % mutation kill rate on `MarketMath` and `FissionMarket` (Vertigo / Gambit)

## Known gaps to close before audit

1. **`FissionMarket` revert paths** — only happy-path swaps are heavily exercised.
   Add tests for: post-expiry swap revert, ZeroAddress on every entry, ZeroAmount,
   InsufficientOutput on slippage, NotInitialized on add-liquidity-before-init.
2. **`MarketMath._getRateAnchor`** when `lastLnImpliedRate` causes `newExchangeRate < IONE`
   — currently no test forces that path.
3. **`PMath.mulWadUp` / `divWadUp` / `sqrt`** — not used yet in production code; only
   `mulWadDown` / `divWadDown` are. When the LP add-liquidity path starts using
   `up` for charging more on imbalanced adds, add coverage.
4. **`SYBase.claimRewards` defaults** — the no-op default is unexercised. SY adapters
   that distribute reward tokens (none yet) will need their overrides covered.
5. **`ActionRouter` slippage rejection paths** — most error branches in trade flows.
