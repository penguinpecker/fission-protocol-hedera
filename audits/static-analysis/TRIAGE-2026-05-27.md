## Delta vs 2026-05-06

24 new `(check, file)` pairs appeared. None are new bug classes — all
fall into the same triage buckets as the baseline (HTS facade, user-
supplied recipient, reentrancy-guard-protected functions). New
locations come from contracts that didn't exist on 2026-05-06:
`FissionPeriphery.sol`, `SaucerSwapLPYieldSource.sol` (v2).

| Check | Files (new) | Triage |
|---|---|---|
| `arbitrary-send-eth` | Gateway, Periphery, Unzap, Zap, SY | Accepted — recipient is user-supplied via `address receiver` on a non-payable transfer; standard pattern, slither can't prove the input is sanitized. |
| `unused-return` | Rewards, RouterV3, Gateway, Lens, MegaZap, Periphery, Zap, SY | Accepted — HTS facade calls (`approve`/`transferFrom`) return bool but revert on real failure, so checking the return is redundant. Same triage as baseline. |
| `incorrect-equality` | Gateway, MegaZap, Periphery, SY | Accepted — `== 0` / `!= 0` is the intended semantics for "no deposit" / "non-empty receipt". |
| `reentrancy-balance` | Gateway, Periphery | Accepted — wrapped in `ReentrancyGuardTransient.nonReentrant` modifier; slither doesn't model OZ's transient guard. |
| `reentrancy-eth` | SY | Accepted — same reason; SY's `sweepHbar` is admin-only and re-entry-guarded. |
| `reentrancy-no-eth` | Periphery | Same — non-reentrant. |
| `unchecked-transfer` | Zap | Accepted — HTS facade pattern, as above. |
| `uninitialized-local` | RewardsMarket | Accepted — local struct initialized via assignment immediately after declaration; slither's CFG doesn't follow. |
| `divide-before-multiply` | RewardsMarket | Accepted — intentional truncation point matches Pendle V2 reference math. |
