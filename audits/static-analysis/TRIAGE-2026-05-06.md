## Slither вЂ” High (2)

Both flag `SY_SaucerSwapV2LP.depositLiquidity`:

```
External call: npm.mint{value: msg.value}(mp)         (line 274)
State write:   positionTokenId = newTokenId           (line 276)
```

- **Guard**: `depositLiquidity` is `external payable nonReentrant whenNotPaused`
  (line 237). Cross-function reentrancy into any function that reads
  `positionTokenId` is blocked by the same guard вЂ” `redeemLiquidity`,
  `harvest`, and `_harvest`'s public entry are all `nonReentrant`.
- **External callee**: `npm` is the SaucerSwap V2 NonFungiblePositionManager
  pinned at `0x...3DDbb9` (Hedera `0.0.4053945`). Non-malicious, audited,
  no public callback hook into the caller.
- **Safe-mint callback**: NPM does emit `onERC721Received` if the recipient
  is a contract; `SY_SaucerSwapV2LP` does not implement the receiver
  interface and the NFT is held against `address(this)` via NPM's internal
  bookkeeping вЂ” no transfer back-hop to us.

**Verdict**: false positive. Same pattern flagged in pass-2 (M-NEW-1 was a
distinct concern about `claimRewards` return-value usage, addressed via
the return-value-only update path).
